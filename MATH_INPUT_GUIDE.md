# 数学I・A・II・B インプット解説書

このノートは、アプリで問題を解く前に頭へ入れておくべき「判断の型」を短くまとめたものです。
狙いは、長い講義を全部見るより先に、解くときの見方を作ることです。

## 使い方

1. まず 1 単元だけ読む
2. その単元の「判断チェック」を声に出す
3. アプリで 3 問だけ解く
4. 間違えたらこのノートへ戻る

## 二次関数

### 最初に見ること

- 軸はどこか
- 定義域はあるか
- 頂点と端点のどれを比べるべきか

### 判断チェック

- 軸は範囲内か
- 上に開くか下に開くか
- 頂点だけでなく端点も見るか

### よくあるミス

- 定義域を忘れる
- 軸の符号を逆にする
- 端点比較を1つしかやらない

## 三角比

### 最初に見ること

- 角度は鋭角か鈍角か
- `sin / cos / tan` のどれか
- 基準角はいくつか

### 判断チェック

- 基本値は分かるか
- 象限で符号が変わるか

### よくあるミス

- 45° と 60° を混同
- 鈍角で符号を落とす

## 場合の数・確率

### 最初に見ること

- 順番は関係あるか
- 全体は何通りか
- 余事象のほうが楽か

### 判断チェック

- 順列か組合せか
- 有利な場合と全体を分けたか
- 重複カウントがないか

### よくあるミス

- 順列と組合せを逆にする
- 分母を取り違える
- 樹形図を省きすぎる

## 指数・対数

### 最初に見ること

- 指数の形に戻せるか
- 底は何か
- 定義域の制約はあるか

### 判断チェック

- `a^x = b` に戻したか
- 対数法則を使う前に意味を確認したか

### よくあるミス

- 底と真数を逆に読む
- 定義域を見落とす

## 微分法

### 最初に見ること

- 求めたいのは傾きか、増減か、極値か
- まず微分する式は何か

### 判断チェック

- 微分してから代入したか
- 符号表まで追うべきか

### よくあるミス

- 代入の順番が逆
- 符号判定を飛ばす

## 数列

### 最初に見ること

- 等差か等比か
- 一般項か和か
- 初項と公差、公比は何か

### 判断チェック

- `n-1` を忘れていないか
- 和の式へ入る前に一般項を確認したか

### よくあるミス

- `n` と `n-1` の混同
- 和の公式の代入ミス

## ベクトル

### 最初に見ること

- 成分で処理できるか
- 内積か位置ベクトルか
- 図形条件を式に直せるか

### 判断チェック

- 対応成分を掛けているか
- 符号を落としていないか

### よくあるミス

- 足し算だけする
- 図の条件を式に翻訳しきれない

## チェス式学習法

アプリでは、同じ `skill_tag` の問題を 3 連続で解くモードを使えます。
これはチェスの「同型局面を連戦して判断を固める」練習と同じ考え方です。

### 解くときの流れ

1. 問題を見る
2. 何の型か決める
3. 判断理由を1つ作る
4. 答える
5. 解説で比較する

### 強くなるコツ

- 正解だけで満足しない
- 「なぜその選択肢にしたか」を残す
- 間違えた型をすぐ3問やる

## 最後に

この解説書は、全部覚えるためのものではなく「見る順番を固定する」ためのノートです。
数学は、知識量だけでなく、見る順番が安定すると一気に伸びます。
